Coverage

Three platforms. One command.

Each collector is independently configurable. Run monitoring for one platform, two, or all three simultaneously.

☁️

Azure Virtual Desktop

Monitors host pools, session hosts, and user sessions via the Azure SDK and Azure Monitor.

  • Host pool status
  • Session host agent version
  • Active / disconnected sessions
  • CPU / memory per host
  • Available hosts for new sessions
  • Scaling plan awareness
  • DefaultAzureCredential auth
  • Multi-subscription
🖥️

Terminal Services (RDS)

Queries on-premises RDSH farms over WinRM using NTLM/Kerberos — no agent required.

  • Session host CPU / mem / disk
  • Active & disconnected sessions
  • RD Connection Broker HA status
  • CAL utilisation (warn/critical)
  • Host uptime / reboot hygiene
  • WinRM NTLM or Kerberos
  • WMI / Get-CimInstance
  • PowerShell remoting
🟡

Citrix VDI (CVAD)

Uses the Citrix Director OData v3 REST API, or falls back to WinRM + Broker PowerShell SDK.

  • Delivery group status
  • Registered vs total VDAs
  • Active / disconnected sessions
  • Delivery Controller health
  • VDA registration state
  • Fault state detection
  • OData API or WinRM fallback
  • Session re-auth on 401
⚡

GPU / vGPU Slices

Per-slice frame buffer, SM utilisation, encoder saturation, and licence ROI analysis via nvidia-smi over WinRM.

  • NVIDIA vGPU B/Q/C/A profiles
  • Per-slice FB used / allocated
  • SM & encoder utilisation
  • Saturated / idle / OK badges
  • Q-profile licence waste detection
  • Slice packing efficiency
  • GPU temp & power draw
  • Per-process GPU attribution

Capabilities

Everything wired up out of the box

No separate agents, no SaaS dependency, no vendor lock-in.

🖥️

Live TUI Dashboard

Full-screen Rich terminal dashboard with live refresh. Three platform panels plus GPU slice panel. Runs anywhere with a terminal — SSH, bastion, local. python -m vdm

📊

Prometheus Exporter

73+ metric samples per scrape across all platforms. Dedicated vGPU slice metrics per instance. Custom registry — won't collide with your app metrics. Binds to 127.0.0.1:9090 by default.

📈

Grafana Dashboard

25-panel importable JSON dashboard. Status columns with colour-mapped value thresholds. Time series for CPU/memory/sessions, gauge for CAL utilisation, bar gauge for unregistered VDAs.

🔔

Alertmanager + Teams

29 firing rules across 4 groups. Native msteams_configs (Alertmanager 0.27+). 9 inhibition rules prevent alert storms. Separate critical / warning / meta channels.

🔍

Best-Practice Analysis

21 checks across AVD, RDS, and Citrix. HA, capacity, performance, version currency, hygiene, licensing. Scored report (0–100 per platform). --analyze exits 2 on CRITICAL — CI-friendly.

⚡

GPU / vGPU ROI Analysis

Flags saturated slices (need bigger profile), idle slices (wasting FB), Q-profile licences on Office users, encoder bottlenecks, and low slice packing efficiency.

🔐

Security-First Design

WinRM defaults to HTTPS (port 5986). ${ENV_VAR} substitution in config — no passwords in YAML. Credentials redacted from all error messages, logs, and notifier payloads. Config file permission check on startup.

🐳

Docker / Container Ready

Included Dockerfile (python:3.12-slim, non-root) and docker-compose.yml with Prometheus + Alertmanager stack. Azure credentials via env vars. --exporter-only mode for sidecar deployments.

GPU Intelligence

The vGPU analysis Citrix Director doesn't show you

Every vGPU slice queried live via nvidia-smi over WinRM. Frame buffer, SM utilisation, encoder saturation, and licence-cost analysis — across AVD, RDS, and Citrix simultaneously.

GRID vPC — Virtual PC

Knowledge workers, Office 365, browser, light 2D. 1–4GB profiles. Lowest per-user licence cost.

CAD, BIM, 3D modelling, professional visualisation. 4–16GB profiles. Highest licence cost.

Quadro vDWS licence (3–5× cost)

AI/ML inference, CUDA compute, simulation. Full tensor core access. Per-concurrent-user pricing.

vCS licence

Single-application delivery without persistent desktop. No OpenGL in desktop context.

Finding Severity What it means Recommended action
Slice saturated ✖ CRITICAL FB >88%, SM >85%, or encoder >85%. Users getting GPU stalls and frame drops. Migrate VM to next-size-up profile (e.g., A16-4Q → A16-8Q).
Encoder saturated ▲ WARNING H.264/HEVC encoder >80%. Display stream to endpoint is throttled. Users see lag before SM saturates. Reduce sessions per slice, or enable HEVC/AV1 on supported clients.
Slice idle/oversized ● INFO FB <20%, SM <5%. Allocated frame buffer sitting empty. Density opportunity. Downsize to smaller profile — run more slices per physical GPU.
Q-profile for light workload ▲ WARNING Quadro vDWS (Q) instance with SM <15% and FB <40%. Paying 3–5× licence premium for Office/browser users. Downgrade to B-profile and switch to GRID vPC licence.
Low slice pack efficiency ● INFO Aggregate FB in use <30% of total allocated across all slices on pGPU. Use smaller profile sizes to pack more users per physical GPU.
GPU process hog ▲ WARNING Single process consuming >60% of GPU engine. Starves other vGPU slices on the same pGPU. Apply NVIDIA vGPU equal-share scheduling, or move heavy user to dedicated C-profile.
pGPU temperature critical ✖ CRITICAL GPU die >90°C. Thermal throttling active — all slices affected. Check server cooling, airflow, and TDP envelope. Reduce active slices temporarily.
Full passthrough (no slicing) ● INFO GPU passed through to single VM. Entire device consumed by one session. Enable vGPU if multiple users share the host. Correct for single-user VDI/CAD workstations.

Get Running

Up in five minutes

Python 3.11+, a config file, and WinRM access to your Windows hosts.

1

Install

shell
git clone https://github.com/tkhemraj/avd-masters.git cd avd-masters pip install -r requirements.txt
2

Configure

config.yaml
# Credentials via env vars — never commit passwords rds: farm_name: "Production RDS Farm" hosts: [rdsh01.corp.local, rdsh02.corp.local] broker_hosts: [rdbroker01.corp.local] winrm_username: "CORP\svc-monitoring" winrm_password: "${RDS_PASSWORD}" use_ssl: true avd: subscription_id: "00000000-0000-0000-0000-000000000000" citrix: director_url: "https://citrix-director.corp.local/Director" username: "CORP\svc-monitoring" password: "${CITRIX_PASSWORD}" prometheus: port: 9090
3

Run

shell
# Live TUI dashboard (default) python -m vdm # Headless + Prometheus exporter only python -m vdm --exporter-only # Best-practice analysis report python -m vdm --analyze # One-shot JSON snapshot python -m vdm --once | jq .rds.license_info
4

Or use Docker Compose

shell
export RDS_PASSWORD="..." export TEAMS_WEBHOOK_CRITICAL="https://outlook.office.com/webhook/..." cd prometheus docker compose up -d # Prometheus on :9091, Alertmanager on :9093
