# Contributing to AVD Masters

Thank you for your interest in contributing to AVD Masters.

We welcome contributions that improve the quality, reliability, and enterprise readiness of the platform.

## Development Philosophy

- **Direct and actionable** — Every feature should help someone make a better decision faster.
- **Professional by default** — Output, documentation, and APIs should be suitable for technical and business stakeholders in regulated environments.
- **Local-first where possible** — Prefer solutions that do not force customers into expensive Azure data services.
- **Evidence-oriented** — Features that help organizations produce defensible artifacts (especially for governance and compliance) are particularly valued.

## Getting Started

1. Fork the repository and create a feature branch.
2. Ensure you have Python 3.11+.
3. Install dependencies: `pip install -r requirements.txt`
4. Run the CLI locally: `python run.py`

## Code Style

- Follow existing patterns for clarity and consistency.
- New modules should include clear docstrings explaining purpose and design decisions.
- Prefer explicit, readable code over clever one-liners.
- Update relevant documentation (README, module docstrings) when behavior changes.

## Pull Requests

- Provide a clear description of the change and the problem it solves.
- Include examples of before/after behavior where helpful.
- Ensure new functionality includes appropriate tests or manual verification steps.
- Be prepared to discuss design trade-offs.

## Reporting Issues

When reporting issues, please include:
- Environment details (Python version, Azure region if relevant)
- Steps to reproduce
- Expected vs actual behavior
- Any relevant logs or output

## Governance and Compliance

Contributions that strengthen support for regulated environments (CMMC 2.0 / NIST 800-171, etc.) are particularly appreciated. Please reference specific controls where relevant.

## Questions

For significant changes, open an issue first to discuss the proposed approach.

We aim to maintain a high standard of quality while moving the platform forward.
